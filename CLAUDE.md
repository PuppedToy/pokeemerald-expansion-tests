# Claude Code Notes

## Files that should almost never be committed

These files are generated by `puppedjs/index.js` (or `analyze.js`) on every run.
Committing them pollutes history with randomizer mutations and corrupts base data.

### Source files mutated by the randomizer/rebalancer
- `src/data/pokemon/species_info/gen_*.h` — base stats, abilities, types (auto-balanced)
- `src/data/pokemon/level_up_learnsets/gen_9.h` — learnset assignments
- `src/data/trainers.party` — trainer party definitions
- `src/data/wild_encounters.json` — wild encounter tables
- `src/starter_choose.c` — starter Pokemon choices
- `src/data/script_menu.h` — TM/HM menu script
- `include/constants/tms_hms.h` — TM/HM constants
- `data/maps/**/scripts.inc` — map trainer scripts
- `data/maps/**/map.json` — map NPC/trainer references

### Puppedjs caches (regenerated from source on each run)
- `puppedjs/pokes.json` — parsed Pokemon data cache
- `puppedjs/level_up_learnsets.json` — parsed learnset cache

### Puppedjs output (fine to keep locally, not useful in history)
- `puppedjs/output/out.html` — analysis HTML viewer
- `puppedjs/output/pokes.js` — Pokemon data for the viewer
- `puppedjs/output/trainers.js` — trainer data for the viewer
- `puppedjs/output/moves.js` — move data for the viewer
- `puppedjs/output/abilities.js` — ability data for the viewer
- `puppedjs/output/wildpokes.js` — wild encounter data for the viewer

Use `analyze.js` to run the pipeline and auto-reset the source files afterward.
