# T-166 — Stock starting bag with catch balls; strip non-Master balls and floor mints from the world

## Context

The player should begin the game already holding a large supply of the common catch balls,
so scattering those same balls around the world (floor pickups, hidden items, NPC gifts) is
redundant. Master Balls are deliberately excluded — they are hand-placed at specific points of
the game and must stay. This is base-content curation on top of which the randomizer runs; the
randomizer's item pools contain **no** Poké Balls (verified in `randomizer/items.js`), and it
never adds balls to floors or gifts, so removing them from the base map data is sufficient for
both the base and randomized ROMs.

## Plan

Three parts:

1. **Starting bag** — add `AddBagItem(ITEM_ULTRA_BALL/QUICK_BALL/TIMER_BALL, 99)` to the new-game
   init in `src/new_game.c` (new games only).
2. **Floor items** — delete every overworld pickup whose item is a Poké Ball other than Master Ball:
   - Item-ball object events in `data/maps/**/map.json` (`OBJ_EVENT_GFX_ITEM_BALL` +
     `trainer_sight_or_berry_tree_id: ITEM_*_BALL`, `Common_EventScript_FindItem`).
   - Hidden items in `data/maps/**/map.json` (`bg_events` `type: hidden_item`, `item: ITEM_*_BALL`).
3. **NPC gifts** — remove every `giveitem ITEM_*_BALL` (non-Master) in `data/maps/**/scripts.inc`,
   including tutorial / minigame / prize handouts (Prof. Birch's starter Poké Balls, Trick House,
   Pokémon Fan Club, lottery, Berry Master, etc.), reworking surrounding script flow (BagFull
   checks, "you got X" messages) so each interaction stays coherent.

"Poké Ball" = item ids 1–3 and 5–27 (`ITEM_POKE_BALL` … `ITEM_CHERISH_BALL`), i.e. all balls
except `ITEM_MASTER_BALL` (4). Excludes non-ball look-alikes (`ITEM_LIGHT_BALL`, `ITEM_SNOWBALL`,
`ITEM_SMOKE_BALL`, `ITEM_IRON_BALL`, `ITEM_STRANGE_BALL`).

### Risks / notes
- Removing an item-ball **object event** shifts positional object-event indices; references use
  symbolic `LOCALID_*` constants (regenerated by the map compiler) so this is safe *unless* a
  script in that map uses a hardcoded numeric object id — checked per map.
- The item randomizer rewrites a curated set of **script-based** field items via anchored labels
  (`ROUTE1xx_BALL` in `scripts.inc`). These are a different mechanism from the object-event
  item balls; take care not to break anchors the randomizer's `replaceAnchored` depends on.
- No local GBA toolchain — the C/map build is only verified in CI or on the builder machine.
  `cd randomizer && npm test` must stay green (the pipeline mutates these files).

Acceptance criteria:
- [ ] New game starts with 99 each of Ultra, Quick, and Timer Balls.
- [ ] No non-Master Poké Ball remains as a floor pickup or hidden item in any map.
- [ ] No non-Master Poké Ball is given by any NPC gift; all reworked scripts remain coherent.
- [ ] Master Balls (floor and any gift) are untouched.
- [ ] No Mint remains as a floor pickup in any map (base or randomized).
- [ ] `cd randomizer && npm test` green.
- [ ] `node scripts/check-tracker.mjs` clean.
